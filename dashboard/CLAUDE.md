# Repo Workflow Rules

Read this once before working in the repo.

## Always Read First

- `AGENTS.md`
- `PROJECT.md`
- `SCOPE.md`
- `TASKS.md`
- `memory/architecture.md`
- `memory/patterns.md`
- `memory/decisions.md`
- `memory/stack-guidance.md`

## Builder Rules

- Work one task at a time.
- Do not skip status transitions.
- Do not mark tasks done.
- Do not edit files outside the task list without stopping and explaining.
- Do not add dependencies casually.
- Do not commit secrets.
- Run verification before handoff.

## QA Rules

- Be skeptical.
- Inspect the diff.
- Verify acceptance criteria.
- Run checks.
- Check security and secrets.
- Approve only with evidence.
- Write exact needs-fix notes when rejecting.
- Update memory after done.

## Process Rules

- Keep `PROJECT.md` current.
- Keep `TASKS.md` progress table current.
- Log real work in `logs/Progress Log.md`.
- Record AI tools used.
- Commit atomically.
